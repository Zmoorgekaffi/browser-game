# Application Documentation

This document contains detailed documentation of the Angular application structure and components.

## Components

### app
- **app.html**: Haupt-HTML-Template der Anwendung
- **app.scss**: Haupt-CSS-Stile der Anwendung
- **app.ts**: Hauptkomponente der Anwendung mit Routing und grundlegender Logik
- **app.routes.ts**: Routenkonfiguration der Anwendung

### Other Components (in src/app/components/)
Die Komponenten folgen dem Muster: name-der-komponente.html, name-der-komponente.scss, name-der-komponente.ts

#### Shared Components
- **redirect-hotspot**: 
  - HTML: redirect-hotspot.component.html
  - SCSS: redirect-hotspot.component.scss
  - TS: redirect-hotspot.component.ts
  - Funktion: Ein interaktiver Hotspot für Navigation mit Klick- und Tastaturunterstützung

- **header**: 
  - HTML: header.html
  - SCSS: header.scss
  - TS: header.ts
  - Funktion: Anzeige von Spielerinformationen (Gold, Rubies, Name, Level)

- **login**: 
  - HTML: login.html
  - SCSS: login.scss
  - TS: login.ts
  - Funktion: Login-Komponente für Charakterauswahl

#### Scene Components
- **village**: 
  - HTML: village.html
  - SCSS: village.scss
  - TS: village.ts
  - Funktion: Hauptansicht des Dorfes mit Navigationshotspots

## Services

### Core Services
- **GameStateService**: 
  - TS: game-state.service.ts
  - Funktion: Zentrale Zustandsverwaltung mit Signals für aktuelle Route und Charakterdaten
  - Speichert aktuelle Charakter-ID in sessionStorage
  - Verwaltet Navigation und Route-Änderungen
  - Lädt und speichert Charakterdaten in localStorage

### Data Services
- **WalletService**: 
  - TS: wallet.service.ts
  - Funktion: Verwaltung von Gold und Rubies mit Signals
  - Methoden: addGold(), spendGold()

- **ProfileService**: 
  - TS: profile.service.ts
  - Funktion: Verwaltung von Spielerprofilinformationen (Name, Level, EXP)
  - Methoden: updateName(), addExp(), levelUp()

- **SkillsService**: 
  - TS: skills.service.ts
  - Funktion: Verwaltung von Skills (Angriff, Verteidigung, Zauber)
  - Methoden: upgradeAttack(), upgradeDefense(), addSpell()

- **InventarService**: 
  - TS: inventar.service.ts
  - Funktion: Verwaltung des Inventars
  - Speichert Items in einem Signal

- **UtilityService**: 
  - TS: utility.service.ts
  - Funktion: Hilfsfunktionen wie Array-zu-Map-Konvertierung und Zufallsindex-Berechnung

### Login Service
- **LoginService**: 
  - TS: login.service.ts
  - Funktion: Verwaltung des Login-Prozesses
  - Erstellt neue Charaktere bei Bedarf
  - Speichert Charakter-ID in sessionStorage

## Models

### GameState Interface
- **game-state.interface.ts**
  - WalletData: { gold: number, rubies: number }
  - SkillsData: { attack: number, defense: number, spells: string[] }
  - ProfileData: { name: string, level: number, exp: number }
  - InventarData: { items: any[] }

## Architecture

### Signal-System
Das Projekt verwendet Angular Signals für reactive State Management:
- Alle Services verwenden signals für ihre Daten
- Zustände werden über Signals geteilt und aktualisiert
- Die Header-Komponente zeigt automatisch aktualisierte Werte an

### Routing
- Routen werden in `app.routes.ts` definiert
- Navigation erfolgt über den Router mit `router.navigate()`
- Route-Änderungen werden über `NavigationEnd` Events verfolgt

### Persistenz
- Charakterdaten werden in localStorage gespeichert
- Aktuelle Charakter-ID wird in sessionStorage gespeichert
- Daten werden beim Login geladen und bei Änderungen gespeichert

### Navigation
- Hotspots sind interaktiv und navigieren bei Klick auf andere Szenen
- Tastaturunterstützung (Enter/Space) für Hotspots
- Backwards-Kompatibilität mit URL-Redirects

### Responsive Design
- Header-Komponente zeigt Spielerinformationen an
- Hotspots sind über CSS positioniert
- Responsive Layouts für verschiedene Bildschirmgrößen

### Security
- Charakter-ID wird in sessionStorage gespeichert (nicht im localStorage)
- Login-Service prüft ob Charakter existiert und erstellt bei Bedarf
- Session-Handling für aktuelle Spieler

## Data Flow

1. Login mit Charakter-ID
2. Charakterdaten werden geladen (aus localStorage oder neu erstellt)
3. Zustand wird in GameStateService gespeichert
4. Header zeigt aktuelle Werte an (Signal-basiert)
5. Navigation zwischen Szenen (z.B. Village, Inventar, Character)
6. Änderungen werden in Signals gespeichert und übertragen
7. Daten werden bei Änderungen in localStorage gespeichert

## Key Features

- **Signal-basiertes State Management**: Modernes Angular State Management mit Signals
- **Reactive UI**: Header aktualisiert sich automatisch bei Datenänderungen
- **Offline-Fähigkeit**: Daten werden lokal gespeichert
- **Responsive Design**: Anpassung an verschiedene Bildschirmgrößen
- **Intuitive Navigation**: Hotspots für einfache Navigation
- **User Experience**: Smooth transitions und reaktive Interaktionen
- **Modular Architecture**: Klare Trennung von Services, Components und Models

## Technical Details

- **Framework**: Angular mit Signals
- **State Management**: Signal-basiertes System
- **Persistence**: localStorage und sessionStorage
- **Routing**: Angular Router
- **Components**: Reusable, modular
- **Services**: Dependency Injection
- **Interfaces**: Typsichere Datenmodelle
