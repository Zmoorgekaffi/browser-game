Wie Stats & Schaden berechnet werden

Vollständige Übersicht über den aktuellen Berechnungscode: alle Basiswerte, jeder Multiplikator, jede Formel — von den Rohattributen bis zum finalen Trefferpunkt im Kampf.

Kernquelle skills.service.ts Kampf fight.service.ts Zauber spells-engine.service.ts

01Überblick

Es gibt keine dedizierte Character-Klasse. Alles läuft über ein einziges state-Signal in SkillsService plus eine berechnete Kette namens combatStats. Diese Kette läuft in vier Schichten, jedes Mal neu berechnet, wenn sich Ausrüstung oder Statpunkte ändern:

1 · Basis

Rohe Grundwerte (Level 1-Defaults)

2 · Ausrüstung

Flache Item-Boni, Attribute, Waffenschaden

3 · Attribut-Skalierung

STR/DEX/INT/VIT/Glück → abgeleitete Kampfwerte

4 · Schrein-Passives

Flat-Boni zuerst, dann Prozent-Boni

Der eigentliche Treffer läuft dann so ab: Grundschaden (Waffenwurf oder Zauberwert) → Schadens-Multiplikator (Stärke/Intelligenz) → bei Elementarschaden Resistenz-Mitigation des Ziels → optional Resolve-Minigame (nur Spieler) → gemeinsame Kampfauflösung (Ausweichen → Krit) → bei physischem Schaden Rüstungs-Mitigation → Energieschild absorbiert zuerst → Rest auf HP.

02Basiswerte (Level 1)

Definiert in skills.service.ts:72-104:

intelligence: 5, dexterity: 5, strength: 5, vitality: 5, luck: 5,
'energy-shield': 0, 'magic-find': 0, armor: 0, hp: 100, mana: 20,
attack: 5, magicAttack: 5, initiative: 10, evasion: 5,
critChance: 5, critDamage: 150, chaosDamage: 0, charisma: 1,
resistances: { fire: 0, cold: 0, lightning: 0, chaos: 0 }

Zusätzlich verwaltet: statPoints (verfügbare Punkte), investedPoints (pro Attribut investiert, Cap 100), unlockedPassives.

03Die Berechnungs-Pipeline

Schicht 2 — Ausrüstung

Für jedes ausgerüstete Item (12 Slots: Kopf, Brust, Bein, Handschuhe, Fußwerk, Accessoire L/R, Kette, Ring L/R, Waffe 1/2, Rücken) werden vier Additionsschritte durchlaufen:

SchrittWas passiertFormel
Flache Statsarmor, energy-shield, magic-find, initiative, evasion, critChance, critDamage, chaosDamage, charismastat += item[key]
Resistenzenfire / cold / lightning / chaos, additivres[e] += item.res[e]
Attributestrength, dexterity, intelligence, vitality, luck — 1:1, ohne Skalierungattr += item.attr
WaffenbereichSchaden-Min/Max aus Waffe + flacher AngriffswertattackMin += dmgMin + flatAtk

Anzeige-Durchschnitt (skills.service.ts:239-242): attack = round((attackMin + attackMax) / 2) — analog für magicAttack.

Schicht 3 — Attribut-Skalierung skills.service.ts:323-341

Läuft über die Gesamt-Attribute (Basis + Ausrüstung + Schrein-Investment). Das ist die zentrale Formelsammlung:

AttributWirkt aufMultiplikator
StärkephysicalDamageMultiplier+0.01 % pro Punkt (1000 Pkt. = +10 %)
Geschickinitiative+2 pro Punkt
IntelligenzmagicDamageMultiplier+0.01 % pro Punkt (analog Stärke)
mana+5 pro Punkt
energy-shield+2 pro Punkt
Vitalitäthp+3 pro Punkt
hp-regeneration+0.5 pro Punkt
GlückcritChance+0.2 pro Punkt (abgerundet)
magic-find+0.5 pro Punkt (abgerundet)

Wichtig

Stärke/Intelligenz wirken nicht mehr flach auf attackMin/Max — diese Ranges kommen jetzt ausschließlich von der Waffe (+ nicht-Waffen-Item-Boni). Stattdessen liefern beide Attribute einen Schadens-Multiplikator, der erst beim tatsächlichen Treffer auf (Waffen-Wurf + Skill-Bonus) angewendet wird — siehe Kampfauflösung und Zauberschaden.

finalStats.physicalDamageMultiplier = 1 + finalStats.strength / 10000;
finalStats.initiative += finalStats.dexterity * 2;
finalStats.magicDamageMultiplier = 1 + finalStats.intelligence / 10000;
finalStats.mana += finalStats.intelligence * 5;
finalStats['energy-shield'] += finalStats.intelligence * 2;
finalStats.hp += finalStats.vitality * 3;
finalStats['hp-regeneration'] += finalStats.vitality * 0.5;
finalStats.critChance += Math.floor(finalStats.luck * 0.2);
finalStats['magic-find'] += Math.floor(finalStats.luck * 0.5);

Schicht 4 — Schrein-Passives skills.service.ts:375-419

Erst werden alle stat-flat-Effekte addiert, danach alle stat-percent-Effekte auf den bereits erhöhten Wert angewendet:

// stat-percent, nachdem alle Flat-Boni angewendet sind:
finalStats.attackMin = Math.round(finalStats.attackMin * (1 + effect.value / 100));
finalStats.attackMax = Math.round(finalStats.attackMax * (1 + effect.value / 100));
// generischer Fall:
finalStats[stat] = Math.round(current + current * (effect.value / 100));

04Level & Statpunkte

Level/XP-Logik liegt entgegen der Erwartung nicht in level-up.service.ts (nur UI-Sichtbarkeit), sondern in profile.service.ts:

const STAT_POINTS_PER_LEVEL = 5;
const MAX_LEVEL = 50;
const BASE_XP_TO_NEXT_LEVEL = 200;
const XP_GROWTH_RATE = 1.09;

xpRequiredForLevel(level) = round(200 * 1.09^(level - 1))

Pro Level-Up gibt es 5 Statpunkte, gedeckelt bei Level 50. Jeder investierte Punkt erhöht das Rohattribut um genau +1 (Cap: 100 investierte Punkte je Attribut). Bei 10/20/30…100 investierten Punkten schaltet ein Schrein-Passiv frei.

05Kampfauflösung fight.service.ts

Initiative — wer beginnt

result = 1d20 + (initiativeSpieler − initiativeMonster); Spieler beginnt wenn result > 10

Ausweichen (Dodge)

Verteidiger würfelt zuerst, Gleichstand gewinnt der Verteidiger:

defenseTotal = 1d20 + verteidigerEvasion
attackTotal  = 1d20 + angreiferLuck
if (defenseTotal >= attackTotal) → komplett ausgewichen, 0 Schaden

Kritischer Treffer

critChancePercent = 1d10 + angreiferCritChance - 5
if (random(0-100) < critChancePercent) {
  critMultiplierPercent = angreiferCritDamage / 2
  finalDamage = round(rawDamage * (1 + critMultiplierPercent / 100))
}

Bei Basis-critDamage = 150 ergibt das +75 % Schaden bei einem Krit.

Asymmetrie

Eingehender Monsterschaden wird vor der Kampfauflösung zusätzlich zufällig reskaliert: scaledDamage = round(damage * (0.75 + random() * 0.25)) — also 75–100 % des Nominalwerts. Spieler-Schaden erhält diese Streuung nicht.

Energieschild & Rüstung combat-roll.util.ts

Der Energieschild absorbiert Schaden 1:1 (min(shield, damage)) bevor HP verringert werden. Rüstung mindert physischen Schaden jetzt aktiv — aber NICHT linear wie Resistenz (siehe Zauberschaden), sondern über eine Diminishing-Returns-Kurve:

mitigationPercent = min(75, armor / (armor + 150) * 100)
finalDamage = round(rawDamage * (1 - mitigationPercent / 100))

Warum nicht linear?

Rüstungs-Werte auf Ausrüstung stapeln sich über 8 Nicht-Waffen-Slots (Kopf/Brust/Bein/Handschuhe/Schuhe/Kette/2×Ring) auf im Schnitt ~29 (Tier 1) bis ~436 (Tier 5) — eine lineare "1 Punkt = 1 %"-Formel wie bei Resistenz würde den 75 %-Deckel schon mit Tier-2-Gear sprengen. Mit der Kurve landet ein Tier-5-Charakter im Schnitt bei ~74 % Mitigation — knapp am Deckel, wie beabsichtigt (siehe Monster-Stats: Monster-Angriffswerte sind bewusst so hoch gesetzt, dass auch bei ~74 % Mitigation noch spürbarer Schaden durchkommt).

Angewendet in FightService.mitigateByArmor() — nur bei damageType = 'physical' (Basisangriff, PHYSICAL_DAMAGE-Skills). Elementarschaden nutzt stattdessen die Resistenz-Mitigation aus SpellsEngineService und überspringt diesen Schritt.

Mana- & HP-Regeneration

manaRegen = 3 + floor(intelligence / 5)

hpRegen = floor(combatStats['hp-regeneration']) — neuer Stat, skaliert mit Vitalität × 0.5/Punkt

Beide werden einmal pro vollständiger Runde angewendet (FightService.endTurn()), HP-Regeneration wird dabei abgerundet.

Monster-Zug

50 % Chance auf einen zufälligen bekannten Zauber, sonst Basisangriff mit flachem monster.attack-Wert (kein Würfelbereich wie beim Spieler).

06Zauberschaden spells-engine.service.ts

Bonuswurf je nach Effekttyp — beim Spieler ein frischer, unabhängiger Wurf über den Kampfwertbereich, beim Monster ein flacher Wert:

bonusAttack = Spieler: rollBetween(attackMin, attackMax) | Monster: monster.attack || 10
bonusMagic  = Spieler: rollBetween(magicAttackMin, magicAttackMax) | Monster: monster.magicAttack || 10

Neu — Waffentyp-Anforderung

Skills können jetzt einen requiredWeaponType tragen (schnitt/stumpf/stich/magie) — z. B. braucht Funke (Feuer) eine magie-Waffe, ein wuchtiger Schlag eine stumpf-Waffe. Geprüft bei jedem Cast (nicht nur beim Ausrüsten) via SkillsService.hasEquippedWeaponType() — SpellsEngineService.castSpell() bricht sonst vor dem Mana-Abzug ab.

Physical Damage

totalDamage = (value + bonusAttack) × physicalDamageMultiplier

Elemental Damage

totalDamage = (value + bonusMagic) × magicDamageMultiplier

Heal

healAmount = spell.effectValues.value

Resolve-Minigame nur Spieler

final = round(value / resolvePoints * correctCount)

Multiplikatoren nur bei Spieler-Casts (siehe Attribut-Skalierung); Monster-Casts nutzen ihre flachen Werte unverändert.

Neu — Element-Resistenz mindert jetzt aktiv

spell.effectValues.element (fire cold lightning chaos) wird gegen die Resistenz DES ZIELS geprüft — linear, 1 Resistenz-Punkt = 1 % Endschaden-Mitigation, gedeckelt bei 75 % (applyResistanceMitigation(), angewendet nach dem Schadens-Multiplikator, vor dem Resolve-Minigame):

totalDamage = round(totalDamage * (1 - min(resistance, 75) / 100))

chaosDamage bleibt weiterhin ungenutzt — nur resistances.{fire,cold,lightning,chaos} wirken.

Beispiel Tier-1-Feuerzauber (firespells_tier1.json): value: 20, requiredWeaponType: "magie", Anforderung intelligence ≥ 10 → Rohschaden (20 + rollBetween(magicAttackMin, magicAttackMax)) × magicDamageMultiplier, dann Resistenz-Mitigation des Ziels, danach Resolve-Rescaling, dann Krit/Dodge.

08Waffen-Qualität / Upgrades

weapon-quality.util.ts — jede Qualitätsstufe addiert +5 %, immer neu berechnet aus dem unveränderlichen Basiswert (kein kumulierter Rundungsfehler):

multiplier = 1 + 0.05 * quality
stats['damage-min']       = round(base-damage-min       * multiplier)
stats['damage-max']       = round(base-damage-max       * multiplier)
stats['magic-damage-min'] = round(base-magic-damage-min * multiplier)
stats['magic-damage-max'] = round(base-magic-damage-max * multiplier)

Ein Upgrade verbraucht die Waffe + 1 passendes Material aus 2 Upgrade-Slots (magie → upgrade-creme, alles andere → grinding).

09Monster-Stats

Monster teilen sich die gleiche Feldstruktur wie der Spieler-Basiszustand, durchlaufen aber keine Attribut-Skalierung — attack/magicAttack/hp/armor/resistances sind einfach die vom JSON vorgegebenen, fest autorisierten Zahlen. Beispiel Goblin (dark-forest.1-10.json, nach dem Rüstungs-/Resistenz-Rebalance):

hp: 49, expReward: 42, mana: 100,
strength: 4, dexterity: 4, vitality: 4, luck: 8,
attack: 23, magicAttack: 12, initiative: 10, evasion: 5,
critChance: 5, critDamage: 150, armor: 1,
resistances: { fire: 1, cold: 1, lightning: 1, chaos: 1 }

XP-Belohnung beim Sieg: expReward ?? 30.

Balance-Skript scripts/balance-sim.mjs

Da Rüstung/Resistenz jetzt aktiv mindern, mussten Monster-Werte neu simuliert werden — reines "Werte hochsetzen" hätte die 35 Düsterwald-Monster entweder trivial oder unbesiegbar gemacht. Kernerkenntnisse aus dem Rebalance:

  • Echte Gear-Stapelung reverse-engineered: 100 zufällige Vollausrüstungen (8 Nicht-Waffen-Slots) je Tier ausgewertet → Ø-Rüstung 29 (Tier 1) bis 436 (Tier 5), Ø-Resistenz/Element 4 bis 61.
  • Monster-Schaden rückwärts aus der Mitigation berechnet: Da ein Tier-5-Charakter im Schnitt ~74 % Rüstungs-Mitigation hat, muss der rohe Monster-Angriff ungefähr das 4-fache dessen sein, was ohne Mitigation nötig wäre, damit am Bracket-Ende noch spürbarer Schaden ankommt (computeBaseAttack()/computeMagicSplit() in balance-sim.mjs).
  • Monster-Luck von Monster-Evasion entkoppelt: Monster-Luck muss mit der (stark gear-getriebenen) Spieler-Evasion mithalten, sonst landen Treffer praktisch nie (1w20+Luck vs. 1w20+Evasion). Monster-Evasion bleibt dagegen niedrig, damit der Spieler seinerseits zuverlässig trifft.
  • Trade-off (bewusst so entschieden): Getunt auf ein forderndes Bracket-Ende (eigenes Tier-Gear) — Bracket-Start (frisch geleveltes, noch altes Gear) bleibt dadurch spürbar härter als vorher, v. a. bei harten Rollen (Brute/Elite/Aberration).

scripts/patch-monster-defense.mjs schreibt die getunten Werte (hp, attack, magicAttack, luck, critChance, armor, resistances) gezielt in die bestehenden Monster-JSONs, ohne von Hand ergänzte Felder (Sprite-Pfade, fight-background, monster-frame) zu überschreiben.

11Verbleibend totgelegt wirkungslos

Nach dem Rüstungs-/Resistenz-Rebalance sind nur noch diese beiden Stats ohne Wirkung im Kampf- oder Zaubercode:

  • chaosDamage — additiv aus Items/Passives gesammelt, aber nie ausgelesen.
  • charisma — vermutlich für ein Dialog-/Sozialsystem vorgesehen, aktuell irrelevant für den Kampf.

armor und resistances.* sind seit dem Rebalance aktiv (siehe Kampfauflösung und Zauberschaden) — nicht mehr Teil dieser Liste.
